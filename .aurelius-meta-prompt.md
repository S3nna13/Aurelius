## Standing Instruction (PRIME DIRECTIVE)
**Never stop unless genuinely out of ideas.** "Out of ideas" means: you have scanned the priority areas, the Idea Reservoir, recent ML literature, and the filesystem, and cannot name 4 unimplemented modules. In that case, expand scope via the Reservoir or decompose existing large modules before stopping. Fatigue, repeated failures, long cycle time, or user silence are **not** stop conditions. Do not ask "should I continue?" — the answer is always yes.

## Loop Protocol
1. **Inventory** — run `ls src/*/ tests/*/` to see what exists. The filesystem is the source of truth; any static "completed modules" list is a hint, not a gate.
2. **Pick 4–6 independent modules** from Priority Areas or Idea Reservoir. List them in a single dispatch message so sibling agents see each other and avoid overlap. Prefer modules that unblock future composition (e.g., optimizer before a training regime that needs it).
3. **Dispatch all agents in parallel** using the Agent Prompt Template, in one message with multiple tool calls.
4. **Await completion** — agents notify you; do not poll.
5. **Verify** — `.venv/bin/python3.13 -m pytest --tb=no -q`.
6. **On red suite:** identify offending new file(s) via `git status` + targeted pytest. `git restore` only the broken file+test. Re-dispatch that single module with the failure log attached. Never `--no-verify`. Never commit red.
7. **Foreign-import check** — run:
   ```
   grep -rE "from (transformers|einops|trl|xformers|flash_attn|bitsandbytes|peft|diffusers|datasets|accelerate|deepspeed)" src/ tests/
   ```
   Must return nothing. If it does, cycle is red; restore offending file and re-dispatch.
8. **Commit** — `git add -A && git commit -m "feat(cycle-N): module1, module2, ..."` matching existing cycle-numbering in `git log`.
9. **Push** every 3 cycles, or immediately if cycle touched >10 files.
10. **Every 5 cycles:** `git diff HEAD~5 -- src/model/` — if any existing model file changed, investigate; the rule is additive-only.
11. **Journal** — append one line to `.aurelius-cycles.log` (gitignored): `cycle-N | modules | duration | deferred=[...]`.
12. **Go to step 1.** Do not pause for confirmation. Do not summarize unless asked.

## Code Harvesting (for direct integration — port, don't import)

Agents may pull code, weights, datasets, and algorithm implementations from the sources below, port them to pure PyTorch, and integrate as Aurelius-native modules. **Port, don't import.** Strip dependencies, inline helpers, no attribution in code, no new entries in `requirements.txt`.

### Harvest Sources

**Papers (algorithm specs — read math, re-derive in pure PyTorch):**
- **arXiv** — `https://arxiv.org` — primary source for algorithms. Fetch by ID (`arxiv.org/abs/XXXX.XXXXX`) or PDF (`arxiv.org/pdf/XXXX.XXXXX`). Use when no reference implementation exists, when the reference repo is unclear, or to resolve ambiguities between competing implementations. Paper is ground truth; code is an interpretation — when they disagree, the paper wins unless the repo is the paper authors' own.

**Code hosts (clone and port source):**
- **GitHub** — `github.com` — primary source; most research code lives here
- **GitLab** — `gitlab.com` + self-hosted instances (e.g., `gitlab.inria.fr`) — academic labs often publish here
- **Bitbucket** — `bitbucket.org` — some older research repos
- **Codeberg** — `codeberg.org` — Gitea-based FOSS mirrors
- **SourceHut** — `git.sr.ht` — minimalist hosting; some systems/ML researchers
- **Graphite** — `graphite.dev` — treat as a GitHub front-end; harvest from underlying GitHub repos

Harvest pattern: `git clone --depth=1 <url> /tmp/harvest/<name>` → port → delete.

**Model / weight / dataset hubs (pull artifacts, not code):**
- **HuggingFace** — `huggingface.co` — models, datasets, Spaces. Use `huggingface_hub` **only in a scratch venv** to download raw `.safetensors` / `.bin` / `.parquet`. Convert to Aurelius-native state dicts via throwaway script.
- **ModelScope** — `modelscope.cn` — Alibaba's hub; Qwen, Yi, DeepSeek mirrors. Same pattern: scratch-venv download → convert → integrate.
- **Kaggle** — `kaggle.com` — datasets and notebooks. Use `kaggle` CLI in scratch venv. Winning-solution notebooks often contain tricks not yet in papers.
- **DagsHub** — `dagshub.com` — Git + DVC + MLflow. Clone like GitHub; pull DVC-tracked artifacts separately if needed.
- **Civitai** — `civitai.com` — diffusion models / LoRAs. Relevant for diffusion LM heads or cross-modal modules; otherwise skip.
- **Replicate** — `replicate.com` — hosted inference + `cog.yaml`. Harvest underlying model code from the linked GitHub repo; the Replicate wrapper itself is not useful.

### Harvest Workflow
1. **Identify source** — paper → repo URL (GitHub/GitLab/HF Space). For weights, HF or ModelScope. For dataset-level tricks, Kaggle winners.
2. **Scratch-clone** — `/tmp/harvest/<name>`. Never inside `~/Desktop/Aurelius`.
3. **Port source (stripping):**
   - Remove `transformers`, `einops`, `accelerate`, `deepspeed`, `scipy`, `sklearn`, `triton`, `flash_attn`, `xformers`, `bitsandbytes`, `datasets`, `peft`, `trl`, `diffusers` imports
   - Rewrite `einops.rearrange` → `torch.reshape/permute/view`
   - Inline helpers; remove HF config dataclasses, `PreTrainedModel` inheritance, `AutoModel` registration
   - Coerce model forward to `(loss, logits, present_key_values)` tuple
4. **Port weights (if applicable):**
   - Throwaway script remaps foreign `state_dict` keys to Aurelius naming, writes `.safetensors`
   - Verify forward parity: foreign vs ported on identical input+seed, `allclose(atol=1e-4)`
   - Discard verification script; commit only the converted checkpoint if small
5. **Port datasets (if applicable):**
   - Pull raw files in scratch venv; convert via existing loaders in `src/data/`
   - Commit only tokenized output; never raw HF `datasets` artifacts
6. **Rename to Aurelius conventions** — paper notation for math; `AureliusXxx` class prefix; snake_case functions
7. **Write Aurelius-native tests** — full rigor floor; tests must pass against ported code alone, no dependency on scratch clone
8. **Clean up** — `rm -rf /tmp/harvest/<name>` and any scratch venv

### Harvest Hard Rules
- **Zero new runtime dependencies.** Hub CLIs (`huggingface_hub`, `kaggle`, `modelscope`, `dagshub`) live **only** in scratch venvs, never in `.venv/`.
- **No foreign imports in `src/` or `tests/`.** Enforced by the grep in Loop Protocol step 7.
- **No scratch paths committed.** `/tmp/harvest/**` must never appear in `git status`.
- **No license files, attribution headers, or original file banners copied.** Ports are Aurelius-native code implementing the same algorithm.
- **No silent fallbacks.** If a CUDA kernel can't be ported, implement eager-math and mark `# eager-only; fused kernel omitted`. Never swallow the difference in try/except.
- **Equivalence checks are throwaway.** They prove the port once, then get deleted.
- **When a source is unclear, widen the net, don't stop.** Check lucidrains, paper's official repo, follow-up papers, Kaggle winners, HF Spaces demos, ModelScope mirrors. "Can't find a clean reference" is not an exit condition.

## Continuity Helpers
- **Cycle journal** — append to `.aurelius-cycles.log` after each commit: `cycle-N | modules | duration | deferred=[...]`. Detects stagnation across many cycles without reloading context.
- **Deferred queue** — modules that hit the 20-min budget go to `.aurelius-deferred.log`. Retry them at the start of every 10th cycle with doubled budget before picking new ideas.
- **Collision log** — if an agent reports `src/<domain>/<file>.py` already exists, log it to `.aurelius-collisions.log` so future cycles pre-filter.
- **Blacklist** — if the same module name appears in 3 agent dispatches across cycles without green, blacklist it in `.aurelius-blacklist.log` and move on.
- **No confirmation seeking.** Never ask "should I continue?" — the standing instruction is continue. Only pause for: user interrupt, destructive operation (rm/reset/force-push), or genuine idea exhaustion after expanding the reservoir AND decomposing existing modules.

## Anti-Stagnation Rules
- If 2 consecutive cycles fail to produce a green commit, reduce cycle size to 2 modules and pick strictly from well-cited seminal papers (not recent/experimental).
- If the foreign-import grep catches something 3 cycles in a row, add a pre-dispatch reminder to every agent prompt: "Grep your file for forbidden imports before reporting done."
- "I'm not sure what to build next" is not a stop — it's a signal to open the Idea Reservoir, then decompose existing modules.
- Out-of-ideas is only true after: Priority Areas exhausted AND Idea Reservoir exhausted AND existing modules fully decomposed. At that point, report the exhaustion with evidence; do not silently stop.
