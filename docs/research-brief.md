Generated by browsing arxiv.org in search of papers that can directly improve Aurelius.
Three categories: inference optimization, agent memory, and training efficiency.

## 1. Speculative Decoding Improvements

### Yggdrasil (arXiv:2512.23858) — NeurIPS 2025
**"Bridging Dynamic Speculation and Static Runtime for Latency-Optimal Tree-Based LLM Decoding"**

Key ideas applicable to Aurelius's speculative decoding (src/inference/speculative*.py):
- **Equal-growth tree structure**: Instead of drafting tokens linearly, use a tree that grows
  evenly across branches — compatible with static computation graphs
- **Latency-aware draft selection**: Choose draft tokens predicted to have the highest
  acceptance probability, not just the top-1 from the draft model
- **Stage-based scheduling**: Separate verification into stages to overlap draft model
  inference with target model verification
- **Result**: Up to 3.98x speedup

Current Aurelius implementation: Uses MTP heads for draft generation (mtp_shared.py) and
standard rejection sampling (speculative_acceptance.py). The Yggdrasil tree approach
could replace the linear draft with a tree, increasing accepted tokens per step.

### Nightjar (arXiv:2512.22420)
**"Dynamic Adaptive Speculative Decoding for Large Language Models Serving"**

Key ideas:
- Dynamically adjust speculation length based on current acceptance rate
- Reduces wasted compute when draft quality drops
- Complements Yggdrasil's tree approach

### Sparse Speculative Decoding (arXiv:2512.21911)
**"Accelerate Speculative Decoding with Sparse Computation in Verification"**

Key ideas:
- During verification, only compute attention for positions that changed
- Skip redundant computation between draft and target model verification
- 1.5-2x speedup on top of existing speculative decoding

## 2. KV Cache Optimization

### PackKV (arXiv:2512.24449)
**"Reducing KV Cache Memory Footprint through LLM-Aware Lossy Compression"**

Key ideas:
- Lossy compression tailored to KV cache data characteristics
- Co-designed compression algorithm and system architecture
- 153% higher memory reduction for K cache, 179% for V cache vs quantization
- 75-171% throughput improvement over cuBLAS mat-vec multiply
- Eliminates decompression overhead entirely

Potential impact on Aurelius: The `gateway/paged_kv_cache.py` and related
serving modules could integrate PackKV's compression.

## 3. Agent Memory Systems

### MemMachine (arXiv:2604.04853)
**"A Ground-Truth-Preserving Memory System for Personalized AI Agents"**

Key ideas:
- Preserves ground truth data while allowing agent personalization
- Structured memory with fact verification
- Designed specifically for personalized AI agents

Potential impact: Aurelius's plugins/memory/ system (19 modules: episodic,
semantic, working, zettelkasten memory) could integrate ground-truth
preservation patterns from MemMachine.

## 4. Training & Post-Training

### PACED (arXiv:2603.11178)
**"Distillation and On-Policy Self-Distillation at the Frontier of Student Competence"**

Key ideas:
- Gradient signal-to-noise ratio (SNR) follows a bell curve over student competence
- Only train on problems at the frontier of student ability
- Wastes less compute than standard distillation

### Apriel-1.5-OpenReasoner (arXiv:2604.02007)
**"RL Post-Training for General-Purpose and Efficient Reasoning"**

Key ideas:
- RL post-training for general-purpose reasoning, not just math/code
- Efficient reasoning with controlled thinking budgets
